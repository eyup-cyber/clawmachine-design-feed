PALETTE USE — CRITICAL
Each scheme carries `void` (page background), `p` (primary accent), `s` (secondary accent), and 7 per-page accents `pg.{dashboard,inventory,analytics,brain,mesh,onboarding,settings}`. Use them as FOREGROUNDS, SURFACES, and BORDERS — not as thin accent stripes on grey. Every neutral traces to the active scheme's void (HSL-shifted lighter or darker), never hardcoded #0E0E0E / #1A1A1A / #666. Two schemes with different voids = different neutrals.

Default starting scheme if you want a lean: #02 DEEP CURRENT — void #060B10, p #00E5A0, s #0078FF. Feel free to diverge per page.

WORDMARK
"clawmachine" always lowercase, Satoshi Bold, single unified baseline, identical kerning, uniform weight. Render as two halves: "claw" in the active scheme's `p`, "machine" in the active scheme's `s`. Palette values only. Never black-on-white or white-on-black.

DATA SHAPES
— deal row is a CSS-grid row with fixed columns: device thumbnail (44px) · vendor chip (120px) · listing title (1fr) · BUY £ (96px) · SELL £ (96px) · ROI pill (72px) · clickthrough ↗ (32px). 12px gap, 48px min row height, baseline-aligned. No column labelled "DEV". Column headers: blank / VENDOR / TITLE / BUY / SELL / ROI / — .
— BUY £ column shows the floor-aggregate price with a small "· N SRC" sub-label (source count). This is what the operator pays. NOT "FLOOR" as a header — "BUY" reads plain English.
— SELL £ column shows the eBay sold-comps median. This is what the operator expects to get when reselling on eBay. NOT "EBAY MED" as a header — "SELL" reads plain English.
— inventory row: device thumbnail + condition pill (MINT / EXCELLENT / GOOD / FAIR) + acquired price + live-comp price + delta pill + action (✓ sold / ↻ relist / ✗ write off)
— KPI card: ALL-CAPS label + big JetBrains Mono number + sparkline or delta
— UK vendors (9): ebay, cex, gumtree, facebook, shpock, preloved, wholesale-clearance, musicmagpie, bs-exchange. Each a palette-coloured chip with lowercase vendor token.

THUMBNAIL GRAMMAR (for when real listing photos are available in production)
Thumbnail container 64×64, palette-surface (scheme-derived).
- When photo exists: photo fills container `object-fit: cover`. Top-left corner = condition grade pill. Top-right corner = device-type chip (20×20 palette `p` fill, 2px outer stroke scheme `s`, 1px inner highlight scheme-lighter, 12×12 icon inside in scheme-contrast colour). Bottom-left = vendor chip. Bottom-right = price overlay. Four corners, four fixed slots, no collision.
- When photo absent (FB / Gumtree often ship without): device icon centres at 32×32 inside the container, palette `p` stroke, scheme-shifted void background. Same card structure; only the thumbnail treatment swaps.
Two-colour logic lives on the CHIP (palette `p` fill vs scheme `s` stroke), not on the icon SVG — keeps the icon readable at any size.

MOTION
cubic-bezier(0.34, 1.56, 0.64, 1) easeOutBack for open-surface transitions. cubic-bezier(0.4, 0, 0.2, 1) for constrained motion. All transitions ≤300ms. Ambient loops ≤2s. Motion carries signal (scan pulse, live counter, log tick) — decoration without signal is out.

ABSOLUTE NO-GOS (hard; everything else is flexible)
1. No translucency, blur, backdrop-filter, opacity <0.9. Apple liquid-glass is the antithesis.
2. No SF Symbols. Lucide Regular outlined SVG (1.5–2px stroke) at 16×16 for nav + UI; hand-authored outlined device silhouettes for thumbnails. Pixelarticons are reference-pool only, never in active UI.
3. No Title Case labels. ALL CAPS tokens or lowercase only.
4. No gamification: no WIN RATE card, XP, streaks, badges, level-ups, leaderboards.
5. No developer-voice UI copy ("In this view we...", "AI-powered", "sophisticated"). Plain labels.
6. No default-visible negative-ROI or dismissed rows. Top positive ROI first; worst-performers as an explicit opt-in section if shown.
7. No Matter, no DM Sans, no style-lab fonts. Satoshi + JetBrains Mono + IBM Plex Sans only.
8. No stock photos, no Unsplash avatars, no Tailwind CDN people, no clipped ovals. Hand-authored device silhouettes only.
9. No CeX or BS Exchange as sell-targets — both are buy-source / floor-aggregate inputs only. Never surface a sell-to-CeX path.
10. No gradients beyond a beveled edge (highlight top-left + shadow bottom-right, solid colours).
11. Radius ≤4px everywhere, spacing from {4, 8, 12, 16, 24, 32} exclusively.
